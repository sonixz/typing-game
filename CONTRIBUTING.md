## ğŸ’» Development Guidelines

### Code Style
- **Keep it simple**: Pure vanilla JavaScript, no frameworks
- **Maintain the aesthetic**: Cyberpunk neon theme (cyan #00ffff, pink #ff0080, purple #8a2be2)
- **Single file architecture**: Keep the game in `index.html` when possible
- **Comment when necessary**: Code should be self-explanatory, but complex logic needs comments
- **Test in multiple browsers**: Chrome, Firefox, Safari

### Commit Messages
- Use descriptive commit messages
- Start with a verb: "Add", "Fix", "Update", "Refactor"
- Example: `Add particle explosion effects for enemy deaths`

### Testing
- Test locally before submitting PR
- Make sure the game still works after your changes
- Test different difficulty levels and scenarios
- Check browser console for errors
